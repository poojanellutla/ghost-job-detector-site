How scoring works

Each role gets a 0–100 Ghost Score from five weighted signals: repost frequency, listing longevity, salary range width, description vagueness, and rapid re-posting. Full methodology and source code are on GitHub.

About this data

This case file blends a 90-day simulated history (to demonstrate the pipeline immediately) with real, ongoing scrapes from public job boards. The scoring algorithm runs identically on both — nothing about the analysis itself is simulated.

Company names and scores reflect posting behavior patterns only (frequency, duration, salary disclosure, description detail) — not a claim about any company's actual hiring intent. Built as a data analytics portfolio project. Source: Python scraper + scoring engine, visualized here and in a companion Power BI dashboard.
